REPORT CARD ENTERPRISE v6.8.1 TO v6.8.2 UPGRADE
PDF AND TEMPLATE STORAGE AUTHORIZATION FIX

BEFORE UPGRADING

1. Keep the confirmed v6.8.1 package unchanged as the rollback baseline.
2. Create and verify an encrypted database and private Storage backup.
3. Record the current GitHub Pages config.js values.
4. Ensure no administrator is replacing templates and no teacher is creating PDFs during deployment.

DATABASE UPGRADE

1. Open the updated 07_schema.sql.
2. Run the appended section beginning with REPORT CARD ENTERPRISE v6.8.2. You may rerun the entire updated 07_schema.sql because its statements are idempotent.
3. Confirm the SQL Editor returns:
   07 SCHEMA v6.8.2 PDF/TEMPLATE STORAGE FIX: PASS
4. Do not run 08_schema.sql. This release continues in 07_schema.sql.

FRONTEND DEPLOYMENT

1. Deploy the contents of GITHUB_PAGES_FRONTEND to the repository root.
2. Preserve the production Supabase URL, Publishable key, and school identity values in config.js.
3. Hard refresh all open browser sessions.
4. Confirm the active service-worker cache is v6.8.2.

REQUIRED ACCEPTANCE TESTS

A. Assigned class teacher
1. Open a Principal-approved, published report in the teacher's assigned home class.
2. Select Create official PDF.
3. Confirm no permission-denied message appears.
4. Confirm the PDF downloads and Download latest PDF becomes available.
5. Refresh the page and confirm the stored PDF remains available.

B. Bulk publication
1. Publish approved reports for one assigned class.
2. Confirm each eligible report is published.
3. Confirm official PDFs are stored or any genuine per-report rendering failure is listed without an authorization error.

C. System Administrator template upload
1. Open Settings, Report Card Templates by Class Range.
2. Upload or replace one valid A4 PDF or DOCX template under 20 MB.
3. Confirm the template is assigned and can be previewed and downloaded.
4. Replace it again and confirm the previous stored object is removed without a permission error.

D. Role regression
1. Sign in as a subject teacher.
2. Confirm the subject teacher cannot publish, create an official PDF, or manage report-card templates.
3. Sign in as Principal and confirm approval remains available but publication and template management remain unavailable.

E. Existing report regression
1. Download an existing stored official PDF.
2. Confirm report viewing, revision history, attendance totals, promotion history, and report workflow are unchanged.

ROLLBACK

If acceptance fails, restore the preserved v6.8.1 frontend and backup. The v6.8.2 SQL changes only restore a required function privilege and replace Storage policies with equivalent, more isolated expressions.
