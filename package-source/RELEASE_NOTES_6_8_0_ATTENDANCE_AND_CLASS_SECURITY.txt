REPORT CARD ENTERPRISE v6.8.0 REUSABLE SCHOOLS EDITION
ATTENDANCE, ALPHABETICAL RECORDS, CLASS SECURITY, AND PROMOTION PLACEMENT

BASELINE
- The confirmed stable production baseline is v6.7.4 Reusable Schools Edition.
- v6.7.4 remains the protected rollback package.

CHANGES
1. Student directory records are ordered alphabetically by the displayed full name, with admission number as a deterministic tie-breaker.
2. Teacher directory records are ordered alphabetically by the displayed full name, with staff number as a deterministic tie-breaker.
3. Principal and linked account lists retain alphabetical name ordering.
4. Class teachers and subject teachers can see students only when the student's current placement belongs to an officially assigned class.
5. Historical enrolments no longer grant visibility to a student who has moved to another class.
6. Student record history shown to teachers is filtered to classes they are authorized to access.
7. Teacher-facing student, report, new-report, and score-import class selectors show only officially assigned classes.
8. Approved or published Term 3 promotion creates or updates the next-class enrolment as the active placement.
9. The previous enrolment is automatically made inactive when promotion is applied.
10. If promotion is withdrawn or no longer valid, the automatic target enrolment is withdrawn and the source enrolment is reactivated.
11. Students below the promotion cutoff remain in their existing class.
12. A new Attendance navigation section is available only to class-teacher accounts.
13. A class teacher can mark attendance only for the class where the account is the assigned class teacher.
14. Daily attendance supports Present, Absent, Late, and Excused statuses.
15. Present and Late count as days present; Absent and Excused do not.
16. Each saved class date counts as one day school opened for that class and term.
17. Future-dated attendance cannot be saved.
18. Term totals automatically synchronize to existing report cards.
19. New or updated report cards automatically receive attendance totals when attendance records exist.
20. Attendance tables are protected through server-side RPC functions and forced Row Level Security.
21. Attendance registers and entries are included in scheduled encrypted backups.
22. The reusable-school generator now packages eight SQL files through 06_schema.sql.

DATABASE
- New permanent upgrade file: 06_schema.sql.
- Existing v6.7.4 installations run only 06_schema.sql.
- Fresh installations run all eight SQL files in order.

FRONTEND
- New Attendance screen for class teachers.
- Automatic attendance fields are read-only in report editing.
- Updated promotion wording reflects the active class placement.

DEPLOYMENT
- Deploy the updated frontend.
- Run 06_schema.sql.
- Redeploy scheduled-backup so attendance data is included in encrypted backups.
- Keep the original v6.7.4 ZIP as the rollback baseline.
