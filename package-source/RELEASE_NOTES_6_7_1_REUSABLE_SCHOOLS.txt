REPORT CARD ENTERPRISE v6.7.1 REUSABLE SCHOOLS EDITION

NEW REUSABLE SCHOOL PACKAGE GENERATOR
- Adds a System Administrator-only GitHub Navigator immediately after Settings.
- Generates a separate fresh complete package for another school without changing the current installation.
- Requires the new school's name and logo.
- Supports a school short name, report-number prefix, user-account email domain, repository name, and optional Supabase browser connection values.
- Rejects Supabase Secret and service-role keys.
- Converts the uploaded logo to a deployment-ready PNG.
- Generates a complete ZIP containing:
  * Seven ordered database SQL files.
  * Three Supabase Edge Functions.
  * A fully branded GITHUB_PAGES_FRONTEND folder.
  * SCHOOL_IDENTITY_SETUP.sql.
  * Fresh installation and backup-recovery guides.
  * Offline backup decryptor.
  * Generated package metadata and deployment README.

BRANDING AND REUSABILITY
- The application now supports package-level school name, logo, short name, report prefix, and email-domain defaults through config.js.
- Generated packages display the specified school identity on login, MFA, sidebar, report PDF, report verification, PWA manifest, browser title, and application icons.
- The generated school identity remains visible even before the identity SQL is applied, while SCHOOL_IDENTITY_SETUP.sql permanently writes it to the separate Supabase database.
- Hard-coded Nipe branding was removed from reusable user-facing workflow text.
- User account email generation now uses school_settings.user_email_domain instead of a fixed @nip.com domain.

DATABASE
- Continues in the existing 05_schema.sql.
- Adds school_settings.user_email_domain with validation.
- Replaces the protected generate_nip_user_email RPC implementation while retaining its legacy function name for API compatibility.
- No 06_schema.sql is required.

DEPLOYMENT
- Upgrade requires the updated 05_schema.sql, frontend deployment, and redeployment of admin-user-management.
- notification-dispatcher and scheduled-backup are unchanged but are included in the fresh package.
