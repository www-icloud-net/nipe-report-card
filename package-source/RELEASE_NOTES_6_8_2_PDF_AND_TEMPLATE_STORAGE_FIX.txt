REPORT CARD ENTERPRISE v6.8.2 REUSABLE SCHOOLS EDITION
REPORT PDF AND TEMPLATE STORAGE AUTHORIZATION FIX
Release date: 22 July 2026
Baseline: Confirmed stable production v6.8.1 Reusable Schools Edition

RESOLVED

1. Official report PDFs
- Restored the authenticated EXECUTE privilege required by Supabase Storage policies for public.can_publish_report(uuid).
- Assigned class teachers and System Administrators can upload and register official PDFs for reports they are authorized to publish.
- Subject teachers remain blocked from publication and official PDF creation.

2. PDF unavailable and PDF not created
- Rebuilt report-pdfs Storage read, insert, update, and delete policies with deterministic bucket-gated CASE expressions.
- Report authorization functions are evaluated only for objects in the report-pdfs bucket.
- Existing published reports without a stored PDF can use Create official PDF successfully after deployment acceptance.

3. Report-card template upload
- Isolated report-card-templates Storage policies from unrelated report-PDF policy evaluation.
- System Administrators can upload, replace, preview, download, and remove approved PDF or DOCX templates.
- Template management remains unavailable to non-administrators.

4. SQL sequence
- The fix continues in 07_schema.sql, following the established schema sequence policy.
- 08_schema.sql is not introduced because 07_schema.sql remains within its practical maintainability limit.

SECURITY

The restored function grant does not grant publication authority. public.can_publish_report(uuid) still returns true only for the System Administrator or the officially assigned class teacher for the report's class. Database RPC validation and Storage RLS both continue to enforce that decision.
